# Plan: Mixed Content (Text + Images) Support for Context Menu Selection

## Goal
When a user selects both text and images on a webpage and uses the context menu "Fact check marked text", currently only the text is sent to the API. The images within the selection are ignored. This plan adds support for extracting and sending both text and images from a mixed selection.

## Architecture Overview

The core challenge is that the Chrome Extension `contextMenus` API only provides `info.selectionText` (plain text string) — it cannot provide images from the selection range. The solution is a two-phase approach:

1. **Phase 1 (background.ts)**: Context menu click saves the selected text as before and opens the popup.
2. **Phase 2 (Selected.svelte / content.ts)**: When the popup opens for a text selection, it queries the content script which uses the DOM Selection API (`window.getSelection()` + `Range.cloneContents()`) to extract both text and any `<img>` elements within the selection range.

## Files to Modify

### 1. `src/TSelectedContent.ts` — Add `SelectedMixed` type

Add a new type that holds an array of content items (text blocks + image blocks in order):

```typescript
export type ContentItem =
  | { type: 'image'; image: string }
  | { type: 'text'; text: string }

export type SelectedMixed = {
  content: ContentItem[]
}
```

Update `SelectedContent` union:
```typescript
export type SelectedContent = SelectedImage | SelectedText | SelectedMixed | undefined
```

Add type guard:
```typescript
export function isSelectedMixed(content: SelectedContent | null): content is SelectedMixed {
  if (!content) return false
  return 'content' in content && Array.isArray(content.content)
}
```

### 2. `src/content.ts` — Extract images from selection range

Add functions to:
- **`getMixedSelection()`**: Uses `window.getSelection()` and `Range.cloneContents()` to get a DocumentFragment of the selection. Iterates the fragment to collect text nodes and `<img>` elements, processes images via `processImage()`, and returns a `SelectedMixed` object.
- **Update `getSelectedContent` handler**: When text is available via `window.getSelection().toString()`, also check for images in the selection range. If images are found, return a `SelectedMixed` object instead of just `SelectedText`.
- **Existing in-page selection modes**: No changes needed — the `enableImageSelect` / `enableTextSelect` modes remain for manual picking.

Key implementation detail: `Range.cloneContents()` creates a copy of the selected DOM. We walk the cloned fragment, for each `<img>` we process it, and for each text node we collect the text. This preserves the order of text and images as they appear in the original selection.

### 3. `src/lib/components/steps/Selected.svelte` — Handle mixed content in popup

- In the `checkPendingContent` function, when `pendingContextMenuText` is found:
  - After setting `selectedContent = { text: pendingText }`, also query the active tab's content script for images via `chrome.tabs.sendMessage({ action: 'getSelectedContent' })`.
  - If the response is a `SelectedMixed`, replace `selectedContent` with the mixed content.
  - This is a best-effort approach (content script may not respond in time, but the text fallback works).
- Alternatively: Send a dedicated message action like `getMixedSelection` that the content script handles specifically by extracting images from the current selection.

- **UI changes**: When `isSelectedMixed(content)`:
  - Display a thumbnail gallery of images with the text below/above
  - Keep the text editable (textarea)
  - Images are shown as thumbnails but not individually editable

### 4. `src/lib/util/checkFact.svelte.ts` — Build mixed content for API

Update the content building logic to handle `SelectedMixed`:

```typescript
if (isSelectedMixed(selectedContent)) {
  content = selectedContent.content.map(item => {
    if (item.type === 'image') {
      return { type: 'image_url', image_url: { url: item.image } }
    }
    return { type: 'text', text: item.text }
  })
}
```

This creates the proper multimodal content array format that LLM APIs expect.

For RAG queries with mixed content: extract text parts only for the query text, same as the current fallback behavior.

### 5. `src/lib/components/steps/Connection.svelte` — Filter endpoints for mixed content

Update `isImageSelected` logic. Currently it only checks `isSelectedImage()`. Add a new derived value:

```typescript
let hasImageContent = $derived(
  isSelectedImage(unifiedStorage.value.selectedContent) || 
  isSelectedMixed(unifiedStorage.value.selectedContent)
)
```

Use `hasImageContent` instead of `isImageSelected` for endpoint filtering, since mixed content also requires an image-capable endpoint.

### 6. `src/lib/components/FactCheck.svelte` — Button disabled logic

The "Check" button disabled state checks `!unifiedStorage.value.selectedContent`. This remains correct since `SelectedMixed` is a valid `SelectedContent`.

## Implementation Order

1. `TSelectedContent.ts` — type definitions and guards
2. `content.ts` — `getMixedSelection()` and updated handler
3. `checkFact.svelte.ts` — handle `SelectedMixed` in API payload
4. `Connection.svelte` — endpoint filtering for mixed content
5. `Selected.svelte` — popup reading mixed content from content script
6. Test with a page containing mixed text+image selection

## Edge Cases

- **Content script not injected**: If `sendMessage` fails (e.g., restricted pages), fall back to text-only.
- **No images in selection**: If `getMixedSelection` finds no images, return a `SelectedText` object instead (backward compatible).
- **Large images in selection**: `processImage()` already resizes to max 512px, which handles size concerns.
- **Multiple images**: Supported naturally by the `content: ContentItem[]` array.
- **Performance**: Image processing is async but only happens for images actually within the selection range.
